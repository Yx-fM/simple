### 1.2 模型结构（统一）

所有实验使用以下基础结构：

```
输入(1x28x28)
-> Conv Block 1 (32 filters, 3x3) + BatchNorm + ReLU + MaxPool + Dropout(0.25)
-> Conv Block 2 (64 filters, 3x3) + BatchNorm + ReLU + MaxPool + Dropout(0.25)
-> Conv Block 3 (128 filters, 3x3) + BatchNorm + ReLU + MaxPool + Dropout(0.25)
-> Global Average Pooling
-> FC (128->256) + BatchNorm + ReLU + Dropout(0.5)
-> FC (256->10)
-> Softmax Output
```

### 1.6 分析问题

**Q1：Adam 和 SGD 的收敛速度有何差异？从实验结果中你观察到了什么？**

答：Adam 收敛速度明显快于 SGD。SGD 需要约 25 个 epoch 才能收敛，而 Adam 在 18 个 epoch 左右即可达到相近的验证准确率。这主要是因为 Adam 使用自适应学习率（通过一阶和二阶矩估计），能够为不同参数计算个性化的学习率，加速收敛。

**Q2：学习率对训练稳定性有什么影响？**

答：学习率过大（>0.01）会导致训练不稳定，出现震荡甚至发散；学习率过小（<0.0001）会导致收敛极慢。0.001 是 Adam 在本任务中的最佳学习率，配合 CosineAnnealingLR 调度器可以进一步稳定训练后期过程。

**Q3：Batch Size 对模型泛化能力有什么影响？**

答：较小的 Batch Size（如 64）通常能带来更好的泛化能力，但训练会更慢且 loss 曲线更噪声。较大的 Batch Size（如 128/256）训练更平稳，但可能导致泛化能力略有下降。本实验选择 Batch Size=64 作为最佳平衡点。

**Q4：Early Stopping 是否有效防止了过拟合？**

答：是的。Early Stopping 在验证 loss 连续 5 个 epoch 无改善时自动停止训练，避免了过度训练。从实验结果看，使用 Early Stopping 的模型（Exp3-5）在验证集上的表现持续优于不使用的情况。

**Q5：数据增强是否提升了模型的泛化能力？为什么？**

答：数据增强显著提升了泛化能力。通过 RandomAffine（旋转，平移，缩放，剪切）和 RandomPerspective 变换，模型见过更多"真实世界"中的变形手写样本，从而在测试集上表现更好。Exp4（使用增强）比 Exp2（不使用）验证准确率高出约 0.3%。

## 附录：实验心得

1. **数据增强的重要性**：MNIST 虽然简单，但真实手写风格多样。强数据增强是提升泛化能力的关键。

2. **BatchNorm 的作用**：在卷积层后加入 BatchNorm 显著加速收敛并稳定训练过程。

3. **交叉验证**：5折交叉验证确保选择最稳定的模型 fold，减少过拟合风险。

4. **Gradio 部署体验**：Gradio 简化了 Web 应用开发，但需要注意输入图像的预处理格式与训练时一致。

5. **模型选择**：并非越复杂的模型越好。简单 CNN + 良好正则化往往优于复杂架构。

6. **输入格式兼容性**：实际使用中发现，MNIST 数据集使用黑底白字格式，但用户手写画板通常使用白底黑字。通过在预处理中添加自动检测和反转逻辑（检测图像平均灰度 > 127 时反转），成功解决了格式不匹配问题，使 Sketchpad 输入的识别准确率从 33% 提升到 100%。
